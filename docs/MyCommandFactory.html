Class MyCommandFactory
Factory class responsible for creating command objects based on command names. Implements the Factory Method Design Pattern.
Syntax
C#
    internal class MyCommandFactory : CommandFactory
Remarks

The Factory Method Pattern allows encapsulating object creation for commands. Clients (the parser) can request a command by name without knowing the exact class.

Benefits include:

  • Centralized creation logic
  • Loose coupling between parser and concrete commands
  • Easy extensibility when adding new commands

Method MakeCommand
Creates and returns a command object based on the provided command name. Acts as the Factory Method.
Syntax
C#
    public override ICommand MakeCommand(string commandName)
Parameters
commandName
Name of the command (case-insensitive). Examples: "circle", "rect", "moveto", "triangle", "square".
Return Value
An instance of ICommand corresponding to the command name. Delegates to BOOSE defaults if the command is unknown.
Remarks

Command Categories:

  • Drawing: moveto, drawto, circle, rect, rectangle, triangle, square
  • Style: pen, pencolour
  • Text: write
  • Control flow: for, endfor, if, else, endif, while, endwhile
  • Variables: int, real, array, poke, peek
  • Methods: method

To add a new command: 1. Create a class inheriting from CanvasCommand or CommandOneParameter/CommandTwoParameters. 2. Add a case in the switch statement. 3. Return a new instance.
